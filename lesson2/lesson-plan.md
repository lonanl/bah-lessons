# Урок 2. CSS на примере карточки сотрудника

## Как пользоваться документом

- Стартовая разметка — в `start.html`. Именно её будем дорабатывать шаг за шагом.
- Итоговый результат — `index.html` + `styles.css` (преподаватель сверяется с ним в конце каждого шага, ученику не показываем до финала).
- Принцип: **всё, что пишем на уроке, остаётся в итоге**. Никаких «а теперь удалим и перепишем».
- Для живого превью удобно запустить любой статический сервер (или просто открыть `start.html` в браузере).

---

## Шаг 0. Что такое CSS (5 минут теории)

### Идея

HTML описывает **что** есть на странице (заголовок, картинка, список). CSS описывает, **как это выглядит** (цвет, размер, расположение). JS — **как оно себя ведёт** (клики, анимации, запросы).

### Что объясняем

- Синтаксис CSS-правила:

```css
селектор {
  свойство: значение;
  свойство: значение;
}
```

- Пример: `h1 { color: red; }` — покрасит все заголовки `h1` в красный.
- CSS можно писать в трёх местах (по возрастанию «правильности»):
  1. Атрибут `style` прямо на теге.
  2. Тег `<style>` в `<head>`.
  3. Отдельный файл `.css`, подключённый через `<link>`.
- Мы пройдём все три способа и поймём, зачем третий лучше первых двух.

### Обсудить с учеником

- Если отключить CSS — как будет выглядеть любимый сайт? (Показать пример: открыть сайт и через DevTools выключить стили.)

## Шаг 1. Первый стиль через атрибут `style`

### Идея

Самый простой способ — написать стиль прямо в HTML через атрибут `style="..."`.

### Что делаем в `start.html`

Добавляем атрибут `style` к трём элементам.

**Заголовок страницы — центрируем:**

```html
<h1 style="text-align: center;">Наша команда</h1>
```

**Должность — делаем серой:**

```html
<p style="color: gray;">Senior Frontend Developer</p>
```

**Абзац «О себе» — меняем размер шрифта:**

```html
<p style="font-size: 16px;">
  Работаю с интерфейсами уже более 8 лет...
</p>
```

### Что объясняем

- `style` — это атрибут, как `href` или `src`. Внутри — CSS-свойства.
- Формат один: `свойство: значение;`. Несколько свойств — через `;`.
- Значения бывают разные:
  - ключевые слова: `center`, `gray`, `bold`;
  - числа с единицами: `16px`, `1.5em`, `50%`;
  - коды цветов: `#2563eb`, `rgba(0,0,0,0.5)`.

### Обсудить с учеником

- Удобно? На первый взгляд — да.
- А если нужно стилизовать 10 одинаковых абзацев? Придётся копировать атрибут. Плохо.
- А если нужно поменять цвет всех заголовков сразу? Правка каждого вручную. Плохо.
- Нужен способ, при котором правило пишется **один раз**, а применяется ко многим элементам.

## Шаг 2. Стили в теге `<style>`

### Идея

Выносим стили из атрибутов в один блок `<style>` в `<head>`.

### Что делаем

1. Удаляем атрибуты `style="..."` с элементов.
2. В `<head>` после `<title>` добавляем блок:

```html
<style>
  body {
    margin: 0;
    padding: 20px;
    font-family: Arial, Helvetica, sans-serif;
    background: #f0f2f5;
    color: #1f2937;
  }

  h1 {
    text-align: center;
    font-size: 28px;
    margin: 20px 0;
  }
</style>
```

### Что объясняем

- Селекторы по тегам: `body`, `h1`, `p` — правило применяется ко всем элементам с таким тегом.
- Свойства для `body`:
  - `font-family` — какой шрифт. Список через запятую: если первого нет, берётся следующий.
  - `background` — цвет фона.
  - `color` — цвет текста (по умолчанию наследуется детьми).
  - `margin`/`padding` — отступы (подробно — на шаге 6).
- Видно, как стиль сразу применился ко всему `<body>` — и к карточке, и ко всему остальному. Это — мощь селектора.

### Что-то пошло не так

- Если попробуем стилизовать `h2` — покрасятся все `h2` (и имя, и в таблице, если были бы). Нужны более точечные селекторы — это дальше.

### Обсудить

- Плюс: правило одно, применяется ко всем `h1`.
- Минус: стили живут в HTML-файле. Если у нас будет 5 страниц — дублирование.

## Шаг 3. Выносим стили в отдельный файл `styles.css`

### Идея

Стили должны лежать отдельно от HTML. Это тот самый «правильный» способ.

### Что делаем

1. Создаём рядом с `start.html` файл `styles.css`.
2. Вырезаем всё содержимое `<style>...</style>` и вставляем в `styles.css` (без самих тегов `<style>`).
3. Вместо `<style>` в `<head>` пишем:

```html
<link rel="stylesheet" href="styles.css" />
```

### Что объясняем

- `<link>` — тег-ссылка в `<head>`. `rel="stylesheet"` говорит браузеру: «это таблица стилей».
- Теперь HTML и CSS живут отдельно. Если создадим вторую страницу — подключим тот же `styles.css` и получим те же стили «бесплатно».

### Обсудить

- В реальных проектах ~95% стилей пишется именно так — в отдельных файлах.

## Шаг 4. Селекторы по `id`

### Идея

Когда нужно стилизовать **конкретный** элемент (а не все такие же на странице), используем `id`.

### Что делаем в `start.html`

Ставим `id="position"` на должность:

```html
<p id="position">Senior Frontend Developer</p>
```

И `id="status"` на `<span>` рядом с аватаром (зелёный кружок, пока невидим):

```html
<span id="status"></span>
```

### Что добавляем в `styles.css`

```css
#position {
  margin: 0 0 4px 0;
  font-size: 16px;
  color: #6b7280;
}
```

К `#status` ещё вернёмся в шаге про `position`.

### Что объясняем

- `#` в CSS — это селектор по `id`.
- Правило `#position` сработает только на том элементе, у которого `id="position"`.
- **Важно:** `id` должен быть уникальным на странице. Два элемента с одинаковым `id` — это ошибка.

### Обсудить

- А что, если нам нужно стилизовать не один элемент, а группу (например, все пункты в списке навыков)? Ставить каждому свой `id`? Перебор. Нужен другой инструмент.

## Шаг 5. Селекторы по классам (+ цвета, фон, обводка)

### Идея

Класс — это «метка», которую можно навесить на сколько угодно элементов. Правило напишем один раз — применится ко всем.

### Что делаем в `start.html`

Добавляем классы на все «строительные блоки» карточки:

```html
<div class="card">
  <div class="header">
    <div class="avatar">
      <img src="..." alt="..." />
      <span id="status"></span>
    </div>
    <div class="info">
      <h2 class="name">Иванова Анна</h2>
      <p id="position">Senior Frontend Developer</p>
      <p class="company">ООО «Пример»</p>
    </div>
  </div>

  <blockquote class="quote">...</blockquote>

  <div class="section">
    <h3>О себе</h3>
    <p class="about">...</p>
  </div>

  <div class="section">
    <h3>Контакты</h3>
    <div class="contacts">
      <a href="mailto:...">anna@example.com</a>
      ...
    </div>
  </div>

  <div class="section">
    <h3>Навыки</h3>
    <ul class="chips">
      <li class="chip">HTML</li>
      <li class="chip">CSS</li>
      ...
    </ul>
  </div>

  <div class="section">
    <h3>Интересы</h3>
    <ul class="chips">
      <li class="chip">Кофе</li>
      ...
    </ul>
  </div>

  <div class="section">
    <h3>Опыт работы</h3>
    <table class="experience">...</table>
  </div>

  <div class="actions">
    <a class="btn" href="mailto:...">Связаться</a>
    <a class="btn" href="#">Скачать резюме</a>
  </div>
</div>
```

Плюс заголовок страницы сверху:

```html
<h1 class="page-title">Наша команда</h1>
```

### Что добавляем в `styles.css`

Правило `h1 { ... }` меняем на `.page-title { ... }` — теперь стилизуем по классу, а не по тегу:

```css
.page-title {
  text-align: center;
  font-size: 28px;
  margin: 20px 0;
}
```

Стилизуем чипы — вводим базовые цвета, фон, обводку:

```css
.chip {
  background: #dbeafe;
  color: #1d4ed8;
  border: 1px solid #93c5fd;
}
```

### Что объясняем

- `.` в CSS — селектор по классу.
- Правило `.chip` применится ко **всем** элементам с `class="chip"`, сколько бы их ни было.
- Форматы цвета:
  - именованные: `red`, `white`, `gray`;
  - HEX: `#2563eb` (3 пары шестнадцатеричных цифр — R, G, B);
  - `rgba(R, G, B, A)` — четвёртое значение (alpha) делает цвет полупрозрачным.
- `border: 1px solid #93c5fd` — обводка: толщина, стиль (solid/dashed/dotted), цвет.

### Обсудить

- У одного элемента может быть сколько угодно классов. Это откроем в следующем шаге.

## Шаг 5.5. Несколько классов на одном элементе

### Идея

Базовый класс задаёт форму, модификатор — вариацию. Переиспользуется форма, меняется только цвет/размер.

### Что делаем в `start.html`

Добавляем цветовые модификаторы к чипам навыков:

```html
<ul class="chips">
  <li class="chip green">HTML</li>
  <li class="chip green">CSS</li>
  <li class="chip green">JavaScript</li>
  <li class="chip blue">React</li>
  <li class="chip blue">TypeScript</li>
  <li class="chip gray">Git</li>
</ul>
```

И модификатор `outline` для чипов интересов:

```html
<ul class="chips">
  <li class="chip outline">Кофе</li>
  <li class="chip outline">Путешествия</li>
  <li class="chip outline">Настольные игры</li>
  <li class="chip outline">Менторинг</li>
</ul>
```

Второй кнопке добавляем модификатор `secondary`:

```html
<a class="btn" href="mailto:...">Связаться</a>
<a class="btn secondary" href="#" download>Скачать резюме</a>
```

### Что добавляем в `styles.css`

Убираем цвет из базового `.chip` (оставляем только форму) и выносим цвет в модификаторы:

```css
.chip.green {
  background: #dcfce7;
  color: #166534;
  border-color: #86efac;
}

.chip.blue {
  background: #dbeafe;
  color: #1d4ed8;
  border-color: #93c5fd;
}

.chip.gray {
  background: #e5e7eb;
  color: #374151;
  border-color: #d1d5db;
}

.chip.outline {
  background: white;
  color: #374151;
  border-color: #d1d5db;
}
```

### Что объясняем

- У элемента может быть сколько угодно классов — через пробел: `class="chip green"`.
- Селектор `.chip.green` (без пробела) означает «элемент, у которого есть **и** `chip`, **и** `green`».
- Это мощный паттерн:
  - `.chip` — общая форма (padding, скругление, размер);
  - `.green` / `.blue` / `.gray` — только цвет.
- Хочешь поменять форму всех чипов сразу — меняешь `.chip`. Хочешь добавить новый цвет — пишешь новое правило `.chip.purple`.

### Обсудить

- Вопрос ученику: «Если написать `.chip .green` (с пробелом) — это то же самое?» (Нет! С пробелом — это «элемент `.green` **внутри** элемента `.chip`».)

## Шаг 6. Box model: `margin`, `padding`, `border`, `border-radius`

### Идея

Любой элемент на странице — это прямоугольник. У этого прямоугольника 4 «слоя»: содержимое → внутренний отступ (`padding`) → рамка (`border`) → внешний отступ (`margin`).

### Что рисуем на бумаге / доске

```
+----------------------------+
|  margin (снаружи)          |
|  +----------------------+  |
|  | border (рамка)       |  |
|  |  +----------------+  |  |
|  |  | padding        |  |  |
|  |  |  +----------+  |  |  |
|  |  |  | content  |  |  |  |
|  |  |  +----------+  |  |  |
|  |  +----------------+  |  |
|  +----------------------+  |
+----------------------------+
```

### Что добавляем в `styles.css`

В самом верху — «сброс»:

```css
* {
  box-sizing: border-box;
}
```

Оформляем карточку:

```css
.card {
  max-width: 560px;
  margin: 20px auto;
  padding: 24px;
  background: white;
  border: 1px solid #e5e7eb;
  border-radius: 16px;
  box-shadow: 0 8px 24px rgba(0, 0, 0, 0.08);
}
```

Чипы — форма:

```css
.chip {
  padding: 6px 12px;
  border-radius: 999px;
  font-size: 14px;
  border: 1px solid transparent;
}
```

Цитата — отступы и полоска слева:

```css
.quote {
  text-align: center;
  font-style: italic;
  color: #4b5563;
  margin: 20px 0;
  padding: 12px 16px;
  border-left: 4px solid #2563eb;
  background: #f9fafb;
}
```

Отделяем секции друг от друга:

```css
.section {
  margin-top: 20px;
  padding-top: 16px;
  border-top: 1px solid #f0f2f5;
}

.section h3 {
  margin: 0 0 10px 0;
  font-size: 16px;
  color: #374151;
}
```

Имя, абзац, компания:

```css
.name {
  margin: 0 0 4px 0;
  font-size: 22px;
}

.company {
  margin: 0;
  color: #6b7280;
  font-size: 14px;
}

.about {
  margin: 0;
  line-height: 1.5;
}
```

Убираем маркеры у списков:

```css
.chips {
  list-style: none;
  padding: 0;
  margin: 0;
}
```

### Что объясняем

- `padding` — внутренний отступ, внутри рамки. Увеличивает «воздух» вокруг содержимого.
- `margin` — внешний отступ, между соседними элементами.
- `border` — рамка. `1px solid #e5e7eb`: толщина, стиль, цвет.
- `border-radius` — скругление углов. `50%` — круг, `999px` — таблетка.
- `box-shadow` — тень. Четыре числа: сдвиг по X, по Y, размытие, разброс. И цвет.
- `box-sizing: border-box` — сделать `width` = весь размер, включая padding и border (по умолчанию — `width` = только content, что неудобно). Пишем `*` — на все элементы.
- `margin: 20px auto` — магия. `auto` слева и справа — элемент центрируется по горизонтали (внутри родителя, у которого есть ширина).

### Обсудить

- Вопрос: «Что будет, если увеличить `padding` у `.card`?» (Карточка раздуется изнутри.)
- «А если увеличить `margin`?» (Карточка отодвинется от соседей.)

## Шаг 7. Flexbox (минимум)

### Идея

Flexbox — способ расположить детей элемента в ряд (или в колонку) с отступами между ними. Одна строчка — и магия работает.

### Что добавляем в `styles.css`

**Шапка карточки** — аватар и инфо рядом:

```css
.header {
  display: flex;
  gap: 20px;
  align-items: center;
  margin-bottom: 20px;
}
```

**Аватар** — фиксированный размер, не даём сжиматься:

```css
.avatar {
  width: 120px;
  height: 120px;
  flex-shrink: 0;
}

.avatar img {
  width: 120px;
  height: 120px;
  border-radius: 50%;
  object-fit: cover;
  border: 3px solid #e5e7eb;
}
```

**Чипы** — в строку, с переносом:

```css
.chips {
  display: flex;
  flex-wrap: wrap;
  gap: 8px;
  list-style: none;
  padding: 0;
  margin: 0;
}
```

**Контакты** — тоже flex:

```css
.contacts {
  display: flex;
  flex-wrap: wrap;
  gap: 16px;
}

.contacts a {
  color: #2563eb;
  text-decoration: none;
}
```

**Блок с кнопками:**

```css
.actions {
  display: flex;
  gap: 10px;
  margin-top: 20px;
}
```

### Что объясняем

- `display: flex` на **родителе** → его прямые дети выстраиваются в ряд.
- `gap: 20px` — расстояние между детьми. Очень удобно, раньше приходилось мучаться с `margin`.
- `align-items: center` — выравнивание детей по поперечной оси (если flex-ряд — то по вертикали).
- `flex-wrap: wrap` — если дети не влезают в строку, переносятся на следующую.
- `flex-shrink: 0` — «не сжимайся». Нужно аватару, чтобы не уменьшался, когда рядом много текста.
- `object-fit: cover` — картинка заполняет контейнер, обрезая лишнее (не искажает пропорции).

### Обсудить

- Вопрос: «Что будет, если у `.chips` убрать `flex-wrap: wrap`?» (Чипы вылезут за карточку в одну строку.)

## Шаг 8. Псевдоклассы

### Идея

Псевдокласс — это «состояние» элемента: наведён курсор, это чётный элемент, в фокусе и т.д. Пишется через двоеточие после селектора: `a:hover`.

### Что добавляем в `styles.css`

Ссылки в контактах:

```css
.contacts a:hover {
  text-decoration: underline;
  color: #1d4ed8;
}
```

Таблица — зебра и подсветка строки под курсором:

```css
.experience {
  width: 100%;
  border-collapse: collapse;
}

.experience th,
.experience td {
  padding: 8px 12px;
  text-align: left;
  border-bottom: 1px solid #e5e7eb;
}

.experience th {
  background: #f9fafb;
  font-size: 14px;
  color: #6b7280;
}

.experience tr:nth-child(even) {
  background: #f9fafb;
}

.experience tr:hover {
  background: #eff6ff;
}
```

Чипы и кнопки реагируют на наведение:

```css
.chip:hover {
  border-color: #2563eb;
}

.btn:hover {
  background: #1d4ed8;
}

.btn.secondary:hover {
  background: #eff6ff;
}
```

Кнопки (ссылки, стилизованные под кнопки):

```css
.btn {
  display: inline-block;
  padding: 10px 20px;
  background: #2563eb;
  color: white;
  text-decoration: none;
  border: 1px solid #2563eb;
  border-radius: 8px;
  font-size: 16px;
  cursor: pointer;
}

.btn.secondary {
  background: white;
  color: #2563eb;
}
```

### Что объясняем

- `:hover` — когда на элемент наведён курсор мыши.
- `:nth-child(even)` — каждый чётный ребёнок внутри родителя. Есть ещё `odd`, `1`, `2n+1` и т.п.
- `border-collapse: collapse` у таблицы — рамки соседних ячеек сливаются в одну.
- `cursor: pointer` — меняет курсор на «палец» (визуально даёт понять, что элемент кликабелен).
- `display: inline-block` у `.btn`: ссылка по умолчанию — `inline`, и `padding` сверху/снизу у неё работает криво. `inline-block` решает.

### Обсудить

- Другие полезные псевдоклассы: `:focus` (когда элемент выбран клавиатурой), `:first-child`, `:last-child`, `:disabled`, `:checked`.

## Шаг 9. `position`

### Идея

Свойство `position` управляет тем, где окажется элемент. С его помощью делаем «зелёный кружок online» поверх аватарки.

### Что добавляем в `styles.css`

```css
.avatar {
  position: relative;
  /* остальное уже есть: width, height, flex-shrink */
}

#status {
  position: absolute;
  bottom: 6px;
  right: 6px;
  width: 18px;
  height: 18px;
  background: #22c55e;
  border: 3px solid white;
  border-radius: 50%;
}
```

### Что объясняем

- Значения `position`:
  - `static` — по умолчанию. Элемент в обычном потоке.
  - `relative` — «в обычном потоке, но могу сдвинуться от своего места через `top/left/right/bottom`». Чаще используется как **якорь для детей** с `absolute`.
  - `absolute` — «выпадаю из потока и позиционируюсь относительно ближайшего предка с `position: relative/absolute/fixed/sticky`. Если такого нет — относительно всей страницы».
  - `fixed` — относительно окна браузера. Не двигается при прокрутке. Для плавающих шапок / кнопок «наверх».
  - `sticky` — гибрид: ведёт себя как `relative`, пока не доскроллим до заданной точки, а потом «прилипает» как `fixed`.
- `top / right / bottom / left` — смещения. У `absolute` это позиция внутри предка.

### Демонстрация

1. Убираем `position: relative` с `.avatar`.
2. Смотрим — зелёный кружок убегает в угол страницы (относительно `body`).
3. Возвращаем `relative` — кружок садится обратно.

### Обсудить

- Вопрос: «Почему у кружка `border: 3px solid white`?» (Чтобы визуально отделить его от картинки, будто он «наложен сверху».)
